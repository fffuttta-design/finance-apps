# FutaFinance「API使用量マネージャー」要件定義

> 最終更新: 2026-07-23 / ステータス: **要件定義（実装未着手）**
> 配置: FutaFinance（`apps/futa_finance`）の1機能として実装する

---

## 1. 目的

**Claude API について「いくら買ったか（購入）」と「いくら使ったか（消費）」を、
FutaFinance の中で1画面で見られるようにする。**

- 購入 = クレジット購入（クレカ引き落とし）＝ **お金が出た額**
- 消費 = 実際にAPIを叩いて減った額 ＝ **どのツールが食ったか**
- この2つの差 ＝ **残っているクレジット残高**

FutaFinance に置く理由：購入側はすでに FutaFinance の事業取引として記録されているため、
「支出」と「その内訳」が同じアプリの中で閉じる。

## 3. 設計方針（Admin APIが使えない前提での作り方）

3つのデータを別々の経路で取り、1画面に合成する。

```
┌─ ①購入（支出） ────────────────────────────┐
│ FutaFinance の既存取引を集計するだけ           │  ← 今すぐ実装できる
│ ・Anthropic (Claude API 追加クレジット)        │
│ ・Anthropic (Claude サブスク=固定費)  ※別枠   │
└──────────────────────────────────────────┘
┌─ ②消費（ツール別）─────────────────────────┐
│ 各アプリが API を呼んだときの usage を自前記録   │  ← 段階導入
│ → 共通の計測エンドポイントへ1行POSTするだけ      │
└──────────────────────────────────────────┘
┌─ ③残高 ────────────────────────────────┐
│ Console の値。自動取得不可 → 手入力 or 推計     │
│ 残高 ≒ 購入累計 − 消費累計 で自動推計できる     │
└──────────────────────────────────────────┘
```

### 3.1 ①購入の集計（実装コスト：小）
FutaFinance の Firestore（事業モード）から、取引先が
`Anthropic (Claude API 追加クレジット)` の取引を月別に合計するだけ。

**サブスク（Claude Max, $110前後/月）は API 課金ではないので別行にして分離表示する。**
混ぜると「APIが高い」と誤読するため。

### 3.2 ②消費の計測（実装コスト：中／段階導入）

**共通の計測受け口を1本だけ作る。**

```
POST /api/ai-usage   （FutaFinance側 or 二村秘書VPS側）
{ "app":"2ch-daihon-maker", "model":"claude-opus-4-8",
  "in":12000, "out":800, "cache_read":9000, "cache_write":0 }
      ↓
Firestore: aiUsage/{YYYY-MM}/{app}/... に加算
```

各アプリ側は、Claude API のレスポンスに必ず入っている `usage` を投げるだけ（数行）。
**失敗しても本体の処理は止めない（fire-and-forget）。**

導入順は消費額の大きい順。**上位2本を入れた時点で87%カバーできる**ので、費用対効果が高い。

| 順 | アプリ | 実装場所 | 効果 |
|---|---|---|---|
| 1 | 2ch-daihon-maker | `core/*.ts` の共通ラッパー1箇所 | 62% |
| 2 | 二村秘書Bot | `core/llm.py` の1箇所 | 25% |
| 3 | shorts-pattern-master | `app/ai.py` | 4% |
| 4〜 | 残り | 順次 | 残り |

※どのアプリも「LLM呼び出しの共通関数」を持っているので、**1アプリあたり実質1箇所の改修**で済む。

### 3.3 円換算
- トークン数 × モデル単価（USD）→ 為替レートで円に。
- レートは FutaFinance の設定値（固定）でよい。将来は自動取得。

| # | 表示 | 内容 |

## 5. 非機能

| 項目 | 要件 |
|---|---|
| 表示金額 | **USD と 円の両方**（ユーザー指定） |
| モード | **事業モード**の機能として実装（API費用は事業経費） |
| 認証 | 既存の FutaFinance 認証をそのまま使う。新しい鍵は不要 |
| 計測の鍵 | 計測エンドポイントは共有シークレット1本で保護（Admin APIキーは使わない＝そもそも無い） |
| 対応端末 | Web / Windows / Android すべて（FutaFinanceの既存3配信先に自動で乗る） |

---

## 6. 実装しないこと

- Claude Code（Maxプラン）の利用量 → サブスクであり API 課金ではない。金額行としてだけ表示
- APIキーの新規発行・無効化 → Console でのみ行う
- 他社AI（Gemini / Groq / OpenAI）の使用量 → 計測エンドポイントの設計だけ汎用にしておき、v2で対応
