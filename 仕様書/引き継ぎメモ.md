### ② 請求書PDFの一括取り込み（✅ 実装完了 v1.0.319）
- 開発中タブ「取込」の下部「請求書PDFを取り込む」から起動（`InvoiceImportScreen`）。
- `data/invoice_extractor.dart`（`InvoiceExtractor`）がPDFバイトを `inline_data`(application/pdf)で
  Geminiに送り、種別(売上/支払)・取引先(発行元/宛先)・日付・税込金額・摘要・支出科目候補を抽出。
- 複数PDFを `file_picker` で選択→1枚ずつ順次解析→プレビューで全項目修正→`TransactionRepository.add` で一括記帳。
- 売上=収入(取引内容を大カテゴリに)/支払・外注=支出(会計科目ドロップダウン・既定「外注費」)。
- モード別カットオフ前の日付はスキップ。`InvoiceExtractor.available` でWeb(キー無)はボタン非表示。
- 残・改善余地: 収入の `incomeSourceId` 紐付けは未対応／インボイス番号(T番号)抽出は未対応／
  並列解析(現状は逐次)。実運用で精度を見て調整。

#### （旧）方針メモ（実装前の確定事項）
- **方針（ユーザー確定）**: アプリ内で**複数PDFを選択 → Gemini が抽出 → プレビュー → 一括記帳**。
  - **売上(入金)請求書 → 収入に記帳 / 支払・外注(制作原価)請求書 → 支出に記帳**（両方対応）。
- 実装メモ:
  - PDFは `file_picker`（既存依存・複数選択 `allowMultiple:true, withData:true`）で取得。
  - GeminiはPDFを直接読める（スキャンPDFも可）。既存の `data/receipt_ocr_cloud.dart`（ReceiptOcrCloud）が
    Gemini呼び出しの参考実装。APIキーは `apps/futa_finance/gemini.key`（gitignore）→ `--dart-define=GEMINI_API_KEY`。
    Webビルド(GitHub Actions)はキー未注入なので、PDF取込ボタンは `ReceiptOcrCloud.available` 等でAndroid/Windows中心に。
  - 抽出項目案: 発行元/取引先・日付・金額(税込)・品目・種別(売上 or 支払)・(支払なら原価カテゴリ候補)。
  - プレビュー画面で一覧確認・修正 → まとめて Transaction 化（収入は収入マスタ、支出は支出カテゴリ/外注費へ）。
  - 証憑PDFの置き場所（参考）: `H:\マイドライブ\①会社運営\1.財務\証憑書類\{月}\` に売上/支払請求書PDF。
    （メモリ `futafinance-初期取り込み.md` 参照。初期取り込みは pdfplumber + 手OCR でやった経緯あり）

## 配信フロー（CLAUDE.md準拠・毎回必須）
1. `apps/futa_finance/pubspec.yaml` の version を +1（X.Y.Z と +B 両方）
2. commit（冒頭 `vX.Y.Z`）→ `git push origin main`（Web自動）
3. Android: apk build（`--dart-define=GEMINI_API_KEY=$(cat gemini.key|tr -d '[:space:]')`）→ `gh release create futa-vX.Y.Z ... --prerelease` → `release/futa-version.json` 更新して push
4. Windows: `cd apps/futa_finance/desktop/Scripts && powershell -ExecutionPolicy Bypass -File build_desktop.ps1 -Publish`
