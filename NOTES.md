## 🔐 Firestore データモデル設計

### futa-finance プロジェクト

```
users/{uid}/
  business/                       ← 事業モード
    transactions/{txId}             ← 1取引1ドキュメント
    config/payments                 ← bankAccounts + creditCards
    config/categories               ← CategoryConfig
    config/subscriptions            ← SubscriptionConfig
    config/income_sources           ← IncomeSourceConfig
    config/checklist                ← ChecklistConfig
    monthly_snapshots/{yyyy-MM}     ← 月初残高
    month_closings/{yyyy-MM}        ← 月末締め
  personal/                       ← 個人モード（同じ構造）
    transactions/...
    config/...
```

設計判断:
- 個別取引は1ドキュメント1取引（数百〜数千件のため）
- 設定系（カテゴリ・支払方法・サブスク等）は config 配下に1ドキュメント
- 月別データは {yyyy-MM} を key に

### Repository アーキテクチャ

```
abstract class XxxRepository {
  static XxxRepository instance = LocalXxxRepository();
  static void useLocal() { ... }
  static void useFirestore(String uid) { ... } // 後で
}

class LocalXxxRepository implements XxxRepository {
  // SharedPreferences ベース（既存）
}

class FirestoreXxxRepository implements XxxRepository {
  // Firestore + offline cache（後で）
}
```

UI 側は `XxxRepository.instance.foo()` のままで、起動時にどちらを使うか切替。

## 🧭 運用ルール: 売上の発生月 vs 入金月（v1.0.75 確定）

事業の売上は「5月分が6月20日入金」のようにタイムラグがある。
v1.0.75 時点で **発生主義シンプル（案A）** を採用。実装ゼロ・ルールだけで運用。

### ルール
1. **売上は発生月の日付で取引登録**（請求書発行/案件完了のタイミング）
2. 金額は確定値があればそれ、無ければ暫定値で OK
3. 入金先（支払方法）は **実際に入金される銀行口座** を指定
4. **入金確定後に金額を編集で上書き**（5月の取引を後から修正）
5. メモ欄に「6月20日入金予定」など追記すると見返しやすい

### 想定される副作用
- 5月の取引で銀行A口座を選んだ場合、**5月時点の「推定残高」**が
  入金前なのに増えて見える → 「実測残高」とズレる
- これは想定内。実測 vs 推定の差は「未入金分」だと理解する
- ホーム画面に「実測 ¥X / 差 +¥Y」表示があるので、その差が大体「未入金分」になる

### 将来困ったら検討する拡張
- 案B: 売掛モデル `Receivable` 新設（発生月/取引先/見込金額/入金予定日/入金済フラグ）
- 案C: `Transaction` に `accrualDate` / `cashDate` の2フィールド
- まずは案A で運用 → 不便を感じたら DevLab で案B プロト → 本番昇格

## 💡 構想メモ（v1.0.61 時点で追加）

### 事業の財務管理を本格化

- **PL（損益計算書）の簡易管理**
  - 月次 / 年次の売上・経費・利益を一画面で確認
  - カテゴリ別の経費を「会計科目」っぽくグルーピング
  - 既存の事業モードの transactions ＋ subscriptions から自動集計でほぼ作れる
  - 出力先: 集計タブの中に「PL ビュー」追加 or 専用画面

- **BS（貸借対照表）の簡易管理**
  - 資産（銀行/現金/電子マネー） − 負債（クレカ未払・借入金）= 純資産
  - 「負債」モデルが現状ない → 借入金マスタを新設する必要あり
  - 月次スナップショットで時系列推移も見たい

- **事業の調子の可視化（ダッシュボード）**
  - 月次売上の前年同月比、移動平均、トレンド
  - 利益率、固定費比率、変動費比率
  - 売掛 / 買掛の残高（将来）
  - "ホーム画面の事業モード時の上半分" に置くイメージ

### 貯金増加の追跡（v1.0.63 時点で要検討）

ユーザーから「貯金増加を追いかける機能、どうするのがいいかな？」と相談。
既存データだけで作れる路線と、新モデルを足す路線の2案。

#### 案A: 既存データから自動計算（実装容易・直感的）
- 月初残高スナップショット（`MonthlySnapshotConfig`）は既に毎月取得済み
- 月初残高の月次推移をグラフ化するだけで「総資産トレンド」が見える
- 月初残高 - 前月月初残高 = その月の純増減
- 12ヶ月の純増減合計 = 直近1年の貯金増加額
- **実装場所**: 集計タブ or ホームに「資産推移」セクション追加

#### 案B: 目標貯金額モデルを追加（モチベ重視）
- 「目標額: ¥3,000,000 / 達成期限: 2027/03」のような目標を設定
- 現在の月初残高 / 目標額 で達成度バー
- 平均月間貯蓄額から「このペースだと達成は X 月」を予測
- 達成 / 未達でホームに緑バッジ or 赤バッジ
- **新モデル**: `savings_goal.dart` (id, name, targetAmount, deadline, baseDate)

#### 案C: 「貯金専用口座」フラグ + 月次推移
- 既存の RegisteredBankAccount に `isSavings: bool` を追加
- isSavings=true な口座だけを集計対象に
- 「貯金口座だけの推移」を別グラフで表示
- 普段使い口座と分離して、純粋な貯金増加が見える
- **実装場所**: 銀行口座編集に「貯金専用」チェックボックス追加

#### 推奨: A + C のハイブリッド
- まず A（既存データだけで月次推移グラフ）を Phase 1 として MVP
- 「貯金専用フラグ」を立てた口座だけのフィルタを Phase 2 で追加
- B（目標額モデル）は需要を見てから検討

### 税金・保険料の予算管理（v1.0.68 時点で相談）

ユーザーから「保険料・税金タブどうつけたらいい？予算して計上していきたい」と相談。
通常の月次支出と混ぜると埋もれる + 不定期支払い（年4回など）に対応したい。

#### 既存モデルの限界
- `Subscription` は月払い/年払いの2択 → 住民税の年4回払い等に非対応
- 通常の `Transaction` だけだと事前見積もりができない

#### 案A: 既存 Subscription を拡張
- `paymentSchedule` を追加（quarterly / semiAnnual / custom など）
- メリット: モデル統一、UI 流用
- デメリット: 肥大化、税金特有の概念（源泉/還付）が乗らない

#### 案B（推奨）: 新モデル `BudgetItem` を新設
```dart
class BudgetItem {
  String id;
  String name;          // "住民税", "国民健康保険"
  BudgetKind kind;       // tax / insurance / other
  int annualAmount;      // 年間予算
  List<ScheduledPayment> schedule;
  // [{month: 6, day: 30, amount: 37500}, {month: 8, ...}]
  String? note;
}
```
- 年額 + 支払日リストで管理
- 実支出（Transaction）と紐付け（budgetItemId）で予実突合
- ホームの「来月の予定支払」セクションに反映可

#### 案C: 既存カテゴリ「税金」「保険」+ 予算機能だけ追加
- `CategoryBudget { major, monthlyBudget }` を新設
- 通常の支出として記録 → カテゴリ別合計が予算と突合
- メリット: 既存フロー流用
- デメリット: 個別税目の追跡できない、不定期支払いに弱い

#### 推奨: 案B + α
- Phase 1: `BudgetItem` モデル + 一覧画面（設定からアクセス）
- Phase 2: ホームに「今月の予定支払」セクション追加
- Phase 3: 取引保存時に「予算項目に紐付け」UI
- Phase 4: 予算 vs 実績の集計タブビュー

**置き場所候補**:
- 「設定 → 税金/保険の予算」ボタンで一覧画面
- 集計タブに「予算 vs 実績」ビュー追加
- ホームの主要セクション化はやり過ぎ（季節物）
